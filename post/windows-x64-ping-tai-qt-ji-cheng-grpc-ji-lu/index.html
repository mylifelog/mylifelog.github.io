windows x64平台qt集成gRPC记录

2020-05-22 / 19 min read

公司的一个项目，为了实现Windows平台跨语言跨进程通信，在初步研究了gRPC、Thrift框架后，因为gRPC采用proto buffers作为通信协议（开源真香），以及Thrift的C++依赖boost过于复杂的原因，决定使用gRPC. 在几经折腾之后，终于成功（2020.5.22）.期间也被很多坑爹教程坑过，现将过程分享如下:

目标：将gRPC 1.28.1版本库集成到qt项目中，成功编译并通信。

1.基本需求：

安装VS2017 Community版(不要与VS2015混装，否则gRPC库添加到qt编译报错)
安装QT 5.9.9版, 编译环境选MSVC-2017-x64
github clone vcpkg 编译生成vcpkg.exe

2.下载编译gRPC：

cd到vcpkg目录，在终端执行 vcpkg install grpc:x64-windows (注意不要grpc:x64-windows-static，否则集成后qt会提示runtimelibrary不匹配)
下载编译的过程很漫长，想要加速下载，可以在终端设置代理,以V2RayN为例，设置:

set http_proxy=http://x.x.x.x:1081 
set https_proxy=http://x.x.x.x:1081

下载好之后会将grpc相关保存到vcpkg的\installed\x64-windows目录如图：

其中，include文件夹是需要qt项目引入的所有头文件，lib文件夹是需要引入的静态文件，tools下面的protobuf和grpc是生成的exe文件，主要用途是将.proto转为某种语言的代码。将这两个文件的exe放到一起，方便使用。

3.编译proto文件(生成cpp,csharp,python三种为例)：

将第2步的exe放到一起后，新建cpp,csharp,python文件夹。新建bat文件，添加如下内容：

protoc -I ./ --grpc_out=./cpp --plugin=protoc-gen-grpc=./grpc_cpp_plugin.exe ./*.proto
protoc -I ./ --cpp_out=./cpp ./*.proto
protoc -I ./ --grpc_out=./csharp --plugin=protoc-gen-grpc=./grpc_csharp_plugin.exe ./*.proto
protoc -I ./ --csharp_out=./csharp ./*.proto
protoc -I ./ --grpc_out=./python --plugin=protoc-gen-grpc=./grpc_python_plugin.exe ./*.proto
protoc -I ./ --python_out=./python ./*.proto

将.proto文件拷至该文件夹，执行bat文件可在对应子文件夹生成对应的代码。

将生成的cpp代码h代码和实例项目的client或者server的代码修改后拷贝到qt项目中。

4.复制文件到QT项目：

将第1步得到的include文件夹拷贝到qt项目文件夹
在qt文件夹下面新建staticlib文件夹，内部再建release，debug两个文件夹。
将第1步得到的 vcpkg\installed\x64-windows\lib\ 内所有文件拷贝到release文件夹
将第1步得到的 vcpkg\installed\x64-windows\debug\lib 内所有文件拷贝到debug文件夹

5.编辑qt内的pro文件：

主要是include grpc的头文件，引入lib库，增加一些编译选项

5.1 引入依赖头文件：

INCLUDEPATH += $$PWD/include
DEPENDPATH += $$PWD/include
INCLUDEPATH += $$PWD/staticlib/debug
INCLUDEPATH += $$PWD/staticlib/release
DEPENDPATH += $$PWD/staticlib/debug
DEPENDPATH += $$PWD/staticlib/release

5.2 引入lib库(除了grpc还需要引入C:\Program Files (x86)\Windows Kits\10\Lib\10.0.17763.0\um\x64下面的WS2_32.lib和AdvAPI32.Lib)

win32:CONFIG(release, debug|release) {
   //release库
    PRE_TARGETDEPS += $$PWD/staticlib/release/cares.lib
    LIBS += -L$$PWD/staticlib/release/ -lgpr
    PRE_TARGETDEPS += $$PWD/staticlib/release/gpr.lib
    LIBS += -L$$PWD/staticlib/release/ -lgrpc
    PRE_TARGETDEPS += $$PWD/staticlib/release/grpc.lib
    LIBS += -L$$PWD/staticlib/release/ -lgrpc_cronet
    ...
    //后面还有很多，省略了，注意新版grpc依赖absl库，所以非常多的lib你都要添加进来
}
else:win32:CONFIG(debug, debug|release) {
   //debug库 同release 只是路径为debug下
}

5.3 添加编译选项：
DEFINES += _WIN32_WINNT=0x600

5.4 执行qmake后即可编译运行
